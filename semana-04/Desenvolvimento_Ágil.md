# Engenharia de Software — Sommerville (Cap. 3)  

## 1. Diferença entre Desenvolvimento Ágil e Desenvolvimento Dirigido por Plano

### Desenvolvimento Dirigido por Plano
- Baseado em planejamento extensivo antes da implementação.
- Requisitos são definidos de forma completa no início do projeto.
- Mudanças são controladas e, muitas vezes, caras.
- Fases bem definidas: requisitos -> design -> implementação -> testes.
- Funciona melhor quando os requisitos são estáveis e bem conhecidos.

**Problema principal:** baixa flexibilidade a mudanças.

---

### Desenvolvimento Ágil
- Baseado em desenvolvimento incremental e iterativo.
- Requisitos evoluem ao longo do projeto.
- Feedback contínuo do cliente é essencial.
- Mudanças são esperadas e bem-vindas.
- Entregas frequentes de software funcional.

**Foco principal:** adaptação rápida e entrega contínua de valor.

---

## 2. Resumo do Método XP (Extreme Programming)

O XP é um dos métodos ágeis mais conhecidos e foca em **qualidade de código e feedback constante**.

### Principais práticas do XP:
- Desenvolvimento em pares (pair programming)
- Testes automatizados constantes (TDD)
- Integração contínua
- Releases frequentes em pequenos incrementos
- Simplicidade no design (fazer o mais simples possível)
- Refatoração constante do código
- Cliente sempre disponível para feedback

### Ideia central:
Melhorar qualidade e reduzir risco através de ciclos curtos e feedback rápido.

## 3. Histórias de Usuário: motivo da criação e funcionamento

### Por que foram criadas?
Histórias de usuário surgiram para resolver problemas de:
- Requisitos excessivamente formais e difíceis de entender
- Falta de comunicação entre cliente e desenvolvedor
- Documentação longa e pouco flexível

Elas simplificam a descrição dos requisitos, tornando-os mais naturais e focados no usuário.

---

### Como funcionam?
Uma história de usuário descreve uma funcionalidade do ponto de vista do usuário final.

Formato típico:

> “Como um [tipo de usuário], eu quero [objetivo] para [benefício].”

### Exemplo:
> “Como professor, eu quero registrar o desempenho do aluno para acompanhar sua evolução.”

### Características:
- Curta e simples
- Foco em valor ao usuário
- Serve como base para conversas, não como especificação completa
- Evolui durante o projeto

---

## 4. O que motivou o surgimento do Desenvolvimento Ágil?

O desenvolvimento ágil surgiu como resposta a problemas dos modelos tradicionais:

- Mudanças constantes nos requisitos dos sistemas
- Longos ciclos de desenvolvimento (software só entregue no final)
- Alta taxa de falhas em projetos grandes e complexos
- Baixa interação entre cliente e equipe técnica
- Dificuldade de prever tudo no início do projeto

### Ideia central da mudança:
Em vez de tentar prever tudo antecipadamente, é melhor **aprender e ajustar continuamente durante o desenvolvimento**.
