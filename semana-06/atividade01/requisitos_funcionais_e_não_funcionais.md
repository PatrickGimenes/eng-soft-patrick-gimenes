# Requisitos do Sistema - Cardápio Virtual de Restaurante

## Requisitos Funcionais (RF)

**RF1** - O sistema deve permitir que o cliente visualize o cardápio digital com categorias de produtos.

**RF2** - O sistema deve exibir detalhes dos pratos, incluindo ingredientes, preço e tempo estimado de preparo.

**RF3** - O sistema deve permitir que o cliente realize pedidos diretamente pelo dispositivo móvel.

**RF4** - O sistema deve permitir a personalização de pedidos (ex: remoção ou adição de ingredientes).

**RF5** - O sistema deve permitir que o cliente acompanhe o status do pedido em tempo real.

**RF6** - O sistema deve disponibilizar um painel para funcionários visualizarem os pedidos recebidos.

**RF7** - O sistema deve permitir que administradores gerenciem o cardápio (adicionar, editar e remover itens).

**RF8** - O sistema deve permitir que clientes avaliem os pedidos após o consumo.

---

## Requisitos Não Funcionais (RNF)

**RNF1** - O sistema deve ser responsivo, permitindo uso adequado em dispositivos móveis, tablets e desktops.

**RNF2** - O sistema deve apresentar tempo de resposta rápido ao carregar o cardápio e processar pedidos.

**RNF3** - O sistema deve garantir disponibilidade durante o horário de funcionamento do restaurante.

**RNF4** - O sistema deve ser de fácil utilização, com interface intuitiva para clientes sem treinamento prévio.

**RNF5** - O sistema deve garantir a segurança dos dados de pedidos e informações dos usuários.

**RNF6** - O sistema deve suportar múltiplos acessos simultâneos sem perda de desempenho significativa.
